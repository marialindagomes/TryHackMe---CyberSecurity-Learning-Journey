## How Websites Work

### Website Components

* HTML creates the structure of a webpage.
* CSS controls the appearance and design.
* JavaScript adds interactivity and dynamic behavior.

### Front-End

The front-end is the part of the website users interact with directly through a browser.

### Back-End

The back-end handles processing, databases, authentication, and server-side logic.

### Static vs Dynamic Websites

* Static websites display the same content to all users.
* Dynamic websites generate content based on user actions and stored data.

### Cybersecurity Relevance

Understanding how websites are built helps identify vulnerabilities and understand how web attacks occur.

## Putting It All Together

### How a Website Loads

1. User enters a URL.
2. DNS converts the domain name into an IP address.
3. Browser sends an HTTP request to the server.
4. The server processes the request.
5. The server returns an HTTP response.
6. The browser renders the webpage.

### Key Components

#### DNS

* Converts domain names into IP addresses.
* Allows users to access websites using easy-to-remember names.

#### HTTP

* Protocol used for communication between client and server.
* Uses requests and responses to transfer data.

#### Web Server

* Hosts website files and applications.
* Processes client requests.

#### Browser

* Requests website resources.
* Displays content to the user.

### Request-Response Cycle

* Client sends request.
* Server processes request.
* Server sends response.
* Browser displays the result.

### Cybersecurity Importance

Knowledge of the complete web process is essential for:

* Web Application Security
* Penetration Testing
* Vulnerability Assessment
* Bug Hunting
* Ethical Hacking
